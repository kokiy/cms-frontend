## 1. 产品范围

### 1.1 本期交付（In Scope）

- **登录认证**：账号密码登录、注册、登录态持久化、401 失效处理
- **文章管理**：列表查询、分页、关键字搜索、状态筛选、创建/编辑、发布配置
- **标签管理**：标签的新增、编辑、删除、文章关联计数展示
- **全局布局**：侧边栏、顶栏面包屑、内容区布局一致性
- **国际化（i18n）**：使用 react-intl 实现中英文切换

### 1.2 本期不做（Out of Scope）

- 多角色权限系统（仅保留"已登录可访问"最小权限模型）
- 文章版本历史、协同编辑、回收站
- 多主题品牌化

### 1.3 用户角色

| 角色   | 权限范围                 |
| :----- | :----------------------- |
| 管理员 | 可访问全部 v1 页面与操作 |

## 4. 功能需求与验收标准

### 4.1 登录页（`/login`）

#### 功能要求

- 表单字段：账号、密码
- 登录成功后：保存 Token 与用户信息，跳转 `/posts`
- 已登录用户访问 `/login`：直接跳转 `/posts`
- 提供跳转到注册页的链接

#### 验收标准（DoD）

| ID  | 验收项                                                           |
| :-- | :--------------------------------------------------------------- |
| L01 | 账号或密码为空时，前端阻止提交并给出字段错误提示                 |
| L02 | 点击提交后按钮进入 loading，请求完成前不可重复提交               |
| L03 | 接口返回 200（业务成功）后，1 秒内完成页面跳转                   |
| L04 | 接口返回 4xx/5xx（业务失败）时展示错误消息，不清空用户已输入账号 |

---

### 4.2 注册页（`/register`）

#### 功能要求

- 表单字段：账号、密码、确认密码
- 注册成功后：自动登录或跳转登录页
- 已登录用户访问 `/register`：直接跳转 `/posts`
- 提供跳转到登录页的链接

#### 验收标准（DoD）

| ID  | 验收项                                                            |
| :-- | :---------------------------------------------------------------- |
| R01 | 账号、密码或确认密码为空时，前端阻止提交并给出字段错误提示        |
| R02 | 密码和确认密码不一致时，前端阻止提交并给出错误提示                |
| R03 | 点击提交后按钮进入 loading，请求完成前不可重复提交                |
| R04 | 接口返回 200（业务成功）后，跳转登录页或自动登录后跳转到 `/posts` |
| R05 | 接口返回 4xx/5xx（业务失败）时展示错误消息，不清空用户已输入账号  |

---

### 4.3 文章管理（`/posts`）

#### 功能要求

- **列表**：分页、关键字模糊搜索、状态筛选（草稿/已发布）
- **编辑器**：Markdown 编辑与预览
- **发布配置**：在右侧 Drawer 完成标签多选、分类选择、发布开关

#### 验收标准（DoD）

| ID  | 验收项                                                   |
| :-- | :------------------------------------------------------- |
| P01 | 列表筛选条件变化后，自动重置到第 1 页并发起查询          |
| P02 | 分页切换可保留当前筛选条件                               |
| P03 | 编辑页存在未保存变更时，离开页面需二次确认               |
| P04 | 发布动作成功后，列表对应文章状态在当前会话内可见一致更新 |
| P05 | 接口错误时，页面保持可操作，不出现白屏或死锁 loading     |

---

### 4.4 标签管理（`/tags`）

#### 功能要求

- 支持新增、编辑、删除标签
- 展示标签关联文章计数

#### 验收标准（DoD）

| ID  | 验收项                                       |
| :-- | :------------------------------------------- |
| T01 | 标签名称为空时禁止提交                       |
| T02 | 标签名称重复时展示后端返回错误并保持当前输入 |
| T03 | 删除标签前必须二次确认                       |
| T04 | 删除成功后列表即时移除目标标签并更新计数展示 |

---

### 4.5 全局布局（Layout）

#### 功能要求

- **侧边栏**：支持折叠/展开
- **顶栏**：面包屑、用户菜单（含退出登录）
- **内容区**：统一 `padding: 24px`，背景使用 `token.colorBgLayout`

#### 验收标准（DoD）

| ID  | 验收项                                           |
| :-- | :----------------------------------------------- |
| G01 | 刷新页面后保留侧边栏折叠状态                     |
| G02 | 点击退出登录后清空登录态并跳转 `/login`          |
| G03 | 所有主页面在 1366x768 与移动端竖屏均无结构性溢出 |

## 5. 国际化规范

### 5.1 技术选型

使用 `react-intl` 作为国际化库，支持：

- ICU 消息格式（复数、性别、时间/日期格式化）
- 运行时语言切换
- TypeScript 类型安全

### 5.2 语言支持

| 语言     | Locale  | 说明 |
| :------- | :------ | :--- |
| 简体中文 | `zh-CN` | 默认 |
| 英文     | `en-US` |      |

### 5.3 文件结构

```
src/locales/
├── index.ts          # 入口、类型定义
├── zh-CN/
│   ├── common.ts     # 通用文案
│   ├── login.ts      # 登录页
│   ├── posts.ts      # 文章管理
│   └── tags.ts       # 标签管理
└── en-US/
    ├── common.ts
    ├── login.ts
    ├── posts.ts
    └── tags.ts
```

### 5.4 使用规范

- 使用 `useIntl()` Hook 或 `<FormattedMessage />` 组件
- **禁止**在组件中硬编码中文字符串
- 消息 Key 命名规范：`[页面].[模块].[描述]`，如 `login.form.username`
- 语言切换后需持久化到 `localStorage`

### 5.5 验收标准（DoD）

| ID  | 验收项                                           |
| :-- | :----------------------------------------------- |
| I01 | 顶栏提供语言切换下拉菜单，支持中英文切换         |
| I02 | 切换语言后页面无刷新，所有文案即时更新           |
| I03 | 刷新页面后保留用户选择的语言偏好                 |
| I04 | 所有 UI 文案均通过 react-intl 渲染，无硬编码文本 |

## 6. API 契约与请求层规范

### 5.1 类型与代码生成

- 通过 `npm run gen:api` 从 OpenAPI/Swagger 生成 `src/client`
- **禁止手写后端 DTO 类型**（即与接口请求/响应实体一一对应的类型）
- 允许手写 UI ViewModel 类型（仅用于前端展示态，不可替代 DTO）

### 5.2 单一请求入口（消除冲突）

- 业务代码必须通过 `src/services` 发起请求
- `src/services` 以生成客户端为底层能力，并在此统一处理：
  - Token 注入
  - 401 清理状态与跳转登录
  - 错误消息标准化（`notification.error`）
- 页面层禁止直接调用裸 Axios 或 `fetch`

### 6.3 数据请求层（TanStack Query）

- 使用 `@tanstack/react-query` 管理所有服务端状态
- Query Key 命名规范：`[实体].[操作].[参数]`，如 `['posts', 'list', { page: 1 }]`
- 统一通过 `src/hooks/use*.ts` 封装查询与变更操作
- 页面层禁止直接调用 `useQuery`/`useMutation`，必须使用封装后的 Hook

**封装示例：**

```ts
// src/hooks/usePosts.ts
export function usePostList(params: PostListParams) {
  return useQuery({
    queryKey: ['posts', 'list', params],
    queryFn: () => postService.list(params),
  })
}
```

### 6.4 错误处理标准

| HTTP 状态码    | 处理策略                   |
| :------------- | :------------------------- |
| 401            | 清空登录态并跳转 `/login`  |
| 403            | 提示无权限，停留当前页     |
| 5xx / 网络错误 | 提示”服务异常，请稍后重试” |

## 7. 状态管理规范

### 7.1 状态分层

| 状态类型   | 管理工具       | 适用场景                      |
| :--------- | :------------- | :---------------------------- |
| 服务端状态 | TanStack Query | 来自 API 的数据、列表、详情   |
| 客户端状态 | Zustand        | UI 本地状态、用户偏好、临时态 |

### 7.2 Zustand Stores

| Store 名称       | 职责范围                           | 持久化 |
| :--------------- | :--------------------------------- | :----- |
| `useAuthStore`   | Token、用户信息、登录态恢复        | 是     |
| `useGlobalStore` | 侧边栏折叠、语言偏好、全局 UI 偏好 | 是     |
| `usePostStore`   | 文章编辑草稿态（未保存变更）       | 否     |
| `useTagStore`    | 标签编辑临时态                     | 否     |

### 验收标准（DoD）

- `auth/global` 支持持久化
- `post/tag` 默认内存态，刷新后可通过接口重建
- 不允许在组件外散落未归档全局状态单例
- **服务端状态必须由 TanStack Query 管理**，禁止放入 Zustand

## 8. 工程质量门禁

### 8.1 Git Hooks（Husky + lint-staged）

- 使用 **Husky** 管理 Git Hooks
- 使用 **lint-staged** 仅对暂存文件执行检查
- **pre-commit Hook**：执行 `lint-staged`
  - 对 `.ts,.tsx` 运行 `oxlint`
  - 对 `.css,.scss,.less` 运行 `stylelint`
  - 对所有支持的文件运行 `prettier --write`
- **commit-msg Hook**：校验 Commit Message 格式

### 8.2 逻辑校验（Oxlint）

- 启用 `oxlint:recommended` 与 `react-hooks/recommended`
- 禁止忽略 `hooks` 依赖规则（如需豁免必须写明原因与 issue 链接）

### 8.3 样式校验（Stylelint）

- 使用 `stylelint-config-standard`
- 使用 `stylelint-order`，遵循 Recess 属性顺序
- 禁止 ID 选择器；嵌套层级不超过 3

### 8.4 格式化（Prettier）

| 配置项          | 值     |
| :-------------- | :----- |
| `semi`          | `true` |
| `singleQuote`   | `true` |
| `printWidth`    | `100`  |
| `trailingComma` | `all`  |

### 8.5 可执行门禁定义

| 阶段      | 门禁要求                                                      |
| :-------- | :------------------------------------------------------------ |
| Commit 前 | lint-staged 自动执行（oxlint + stylelint + prettier）         |
| CI        | 必须通过 build 与关键路径冒烟测试（登录、文章查询、标签新增） |
| 合并      | 任一门禁失败：禁止合并                                        |

## 9. 性能与体验指标

| 指标项                        | 目标值 / 要求                         |
| :---------------------------- | :------------------------------------ |
| 提交按钮状态                  | 请求期间必须为 loading 且不可重复点击 |
| 骨架屏触发阈值                | 预估加载超过 300ms 时使用 `Skeleton`  |
| 列表首屏可交互时间（TTI）     | 2s 内（开发环境不计）                 |
| 列表操作（分页/筛选）反馈时间 | 1s 内有可感知反馈（loading 或骨架）   |
