# 开发规范

## 6.1 API 契约与请求层

- 通过 `npm run gen:api` 从 OpenAPI/Swagger 生成 `src/client`
- **禁止手写后端 DTO 类型**
- 业务代码必须通过 `src/services` 发起请求
- 页面层禁止直接调用裸 Axios 或 `fetch`

**错误处理标准**：

| HTTP 状态码    | 处理策略                   |
| :------------- | :------------------------- |
| 401            | 清空登录态并跳转 `/login`  |
| 403            | 提示无权限，停留当前页     |
| 5xx / 网络错误 | 提示"服务异常，请稍后重试" |

## 6.2 TanStack Query 规范

- 使用 `@tanstack/react-query` 管理所有服务端状态
- Query Key 命名规范：`[实体].[操作].[参数]`，如 `['posts', 'list', { page: 1 }]`
- 统一通过 `src/hooks/use*.ts` 封装查询与变更操作
- 页面层禁止直接调用 `useQuery`/`useMutation`，必须使用封装后的 Hook

## 6.3 Git 提交规范

项目使用 Commitlint 校验提交信息，格式如下：

```
<type>(<scope>): <subject>
```

Type 可选值：`feat`、`fix`、`docs`、`style`、`refactor`、`test`、`chore`

## 6.4 Prettier 配置

| 配置项          | 值     |
| :-------------- | :----- |
| `semi`          | `true` |
| `singleQuote`   | `true` |
| `printWidth`    | `100`  |
| `trailingComma` | `all`  |

## 6.5 工程质量门禁

| 阶段      | 门禁要求                                              |
| :-------- | :---------------------------------------------------- |
| Commit 前 | lint-staged 自动执行（oxlint + stylelint + prettier） |
| CI        | 必须通过 build 与关键路径冒烟测试                     |
| 合并      | 任一门禁失败：禁止合并                                |

## 6.6 性能与体验指标

| 指标项                        | 目标值 / 要求                         |
| :---------------------------- | :------------------------------------ |
| 提交按钮状态                  | 请求期间必须为 loading 且不可重复点击 |
| 骨架屏触发阈值                | 预估加载超过 300ms 时使用 `Skeleton`  |
| 列表首屏可交互时间（TTI）     | 2s 内（开发环境不计）                 |
| 列表操作（分页/筛选）反馈时间 | 1s 内有可感知反馈                     |

## 6.7 发布阻断项

> **注意**：以下任一问题存在时，禁止发布

- [ ] 登录后无法访问 `/posts`
- [ ] 401 无法正确回到 `/login`
- [ ] 文章发布后列表状态不一致
- [ ] 标签新增/删除出现数据错乱或重复
- [ ] 任一质量门禁未通过
